# 08 · 开发指南

> 给新加入项目的开发：环境怎么搭、代码怎么写、测试怎么跑。

---

## 一、本地开发模式选择

| 模式 | 适用 | 启动方式 |
|------|------|---------|
| **A. 全 Docker** | 演示 / 简单跑通 | `docker compose up -d` |
| **B. 仅基础设施 + 本地代码** | 推荐：开发后端 / 前端 | `docker compose -f docker-compose.infra.yml up -d` + 本地启动 |
| **C. 全本地** | 本机已有 Milvus | 手动 |

下面以 **模式 B** 为主介绍。

---

## 二、环境准备

### 2.1 必装工具

| 工具 | 版本 | 安装 |
|------|------|------|
| Python | 3.11.x（≥3.11, <3.13）| [python.org](https://www.python.org/downloads/) 或 `uv python install 3.11` |
| Node.js | 22.x | [nodejs.org](https://nodejs.org/) |
| uv | 最新 | `pip install uv` 或 [astral.sh/uv](https://docs.astral.sh/uv/) |
| pnpm | 11.x | `corepack enable && corepack prepare pnpm@11.1.2 --activate` |
| Docker | ≥ 24 | Docker Desktop |

> ⚠️ Python 3.13 有兼容问题（部分 LangChain 依赖未更新），坚持用 3.11。

### 2.2 拉代码

```bash
git clone <repo-url> project-01
cd project-01
```

### 2.3 配置 `.env`

```bash
cp .env.example .env
# 编辑 .env：
#   DASHSCOPE_API_KEY=sk-xxx          # 必填
#   TAVILY_API_KEY=tvly-xxx           # 可选（不填会优雅降级）
```

### 2.4 启基础设施

```bash
docker compose -f docker-compose.infra.yml up -d
```

启动 etcd + minio + milvus 三件套，端口 `19530` 对外暴露给本地后端。

## 三、后端开发

### 3.1 装依赖

```bash
cd backend

# uv 自动装 Python 3.11 + 所有依赖
uv sync
```

### 3.2 启动服务

```bash
# 直接命令
uv run uvicorn app.main:app --reload --host 0.0.0.0 --port 8800

# 或用脚本（PowerShell）
./scripts/dev.ps1
```

访问：
- API：http://localhost:8800/api/v1/health
- 文档：http://localhost:8800/docs

### 3.3 目录速记

```
backend/app/
├── api/v1/           # 路由：health/chat/documents/rag/agent
├── core/             # config, logger
├── schemas/          # Pydantic 模型（chat/document/agent）
├── services/         # 业务
│   ├── llm/          #   chat_model, embedding, chains
│   ├── rag/          #   parser/splitter/vector_store/...
│   └── agent/        #   state/tools/graph
└── main.py           # FastAPI app
```

### 3.4 测试脚本

```bash
# 测 LLM 连通
uv run python scripts/test_qwen.py

# 测 API
uv run python scripts/test_api.py

# 端到端 RAG
uv run python scripts/test_rag.py

# 端到端 Agent
uv run python scripts/test_agent.py

# 测 Tavily
uv run python scripts/test_tavily.py

# 重置 Milvus collection
uv run python scripts/reset_milvus.py

# RAG 评估
uv run python scripts/eval_rag.py
```

### 3.5 代码风格

- 用 `ruff` 做 lint + format（已配置 `line-length=100`）
- 类型注解尽量加（FastAPI / Pydantic 受益）
- docstring 写"为什么"而不是"是什么"

```bash
# 检查
uv run ruff check app/

# 自动修
uv run ruff check --fix app/
uv run ruff format app/
```

### 3.6 添加新依赖

```bash
# 加运行依赖
uv add somepackage

# 加开发依赖
uv add --dev pytest-mock
```

会自动更新 `pyproject.toml` + `uv.lock`。

## 四、前端开发

### 4.1 装依赖

```bash
cd frontend
pnpm install
```

### 4.2 启动 dev server

```bash
pnpm dev
```

访问：http://localhost:3300

### 4.3 目录速记

```
frontend/src/
├── app/             # Next.js App Router
│   ├── layout.tsx
│   └── page.tsx     # 主页（直接渲染 ChatPanel）
├── components/
│   ├── chat/        #   ChatPanel + MessageBubble + AgentTrace + ChatInput
│   └── knowledge/   #   DocumentList + UploadButton
├── store/
│   └── chat.ts      # Zustand 状态
└── lib/
    ├── api.ts       # HTTP / SSE 封装
    ├── types.ts     # 与后端对齐的类型
    ├── config.ts    # API base URL
    └── utils.ts     # cn() 等
```

### 4.4 改 API 端点

`src/lib/config.ts`：

```typescript
export const API_BASE_URL =
  process.env.NEXT_PUBLIC_API_BASE_URL || 'http://localhost:8800';
```

构建时：
```bash
NEXT_PUBLIC_API_BASE_URL=https://api.example.com pnpm build
```

### 4.5 调试 SSE 流

打开浏览器 DevTools → Network → 找到 `agent/run/stream` → 点 EventStream 标签，可看到逐条事件。

### 4.6 类型同步

后端 schema 改了，前端 `src/lib/types.ts` 也要同步改（手工，没自动生成）：

```typescript
// 与后端 app/schemas/document.py: Citation 对齐
interface Citation {
  chunk_id: string;
  document_id: string;
  document_name: string;
  content: string;
  score: number;
  chunk_index: number;
}
```

## 五、典型开发场景

### 5.1 新增一个 API 端点

1. **schema**：在 `backend/app/schemas/` 加 Pydantic 模型
2. **service**：在 `backend/app/services/xxx/` 写业务逻辑
3. **route**：在 `backend/app/api/v1/xxx.py` 加路由
4. **聚合**：在 `backend/app/api/v1/__init__.py` 注册 router
5. **前端**：在 `frontend/src/lib/api.ts` 加 client，types.ts 加 type
6. **测**：起后端 + curl / 起前端跑通

### 5.2 新增 Agent 工具

```python
# backend/app/services/agent/tools.py

@tool
async def my_new_tool(param: str) -> str:
    """
    工具的描述（LLM 用这个决定调不调）。
    适用场景：...
    输入：...
    """
    # 实现
    return "结果文本"

# 注册
ALL_TOOLS = [search_knowledge_base, web_search, calculator, get_current_time, my_new_tool]
```

LangGraph 自动支持，无需改图。

### 5.3 调整 RAG 配置

`backend/app/services/rag/retrieval.py: _default_config()`：

```python
def _default_config() -> RagConfig:
    return RagConfig(
        top_k=5,
        candidate_k=20,
        use_bm25=True,
        use_rerank=True,
        use_multi_query=False,   # ← 想试 Multi-Query 改 True
        use_hyde=False,
    )
```

或者 API 层让前端控制，每次请求带不同配置。

### 5.4 改 Prompt

| Prompt | 文件 |
|--------|------|
| RAG | `backend/app/services/rag/retrieval.py: RAG_SYSTEM_PROMPT` |
| Agent | `backend/app/services/agent/graph.py: AGENT_SYSTEM_PROMPT` |
| Multi-Query | `backend/app/services/rag/query_rewrite.py: MULTI_QUERY_PROMPT` |
| HyDE | `backend/app/services/rag/query_rewrite.py: HYDE_PROMPT` |

## 八、Git 工作流（建议）

```
main  ←─── PR ─── feat/xxx
       ←─── PR ─── fix/xxx
```

约定：

- 分支命名：`feat/agent-streaming` `fix/milvus-load` `docs/api`
- Commit 信息：动词在前
  - `feat: support multi-query rewrite`
  - `fix: handle empty BM25 corpus`
  - `docs: update RAG module guide`
- PR 模板（建议）：
  ```
  ## 改动
  - 一句话总结
  
  ## 影响范围
  - [ ] 后端
  - [ ] 前端
  - [ ] 文档
  
  ## 自测
  - 跑了哪些 test_xxx.py / 手测的步骤
  ```

## 九、常见踩坑

| 现象 | 排查 |
|------|------|
| `uv sync` 慢 | 已配清华源；如还慢检查代理 |
| Pydantic v1 vs v2 | 项目锁定 v2；`Field(min_length=1)` 不是 `min_items` |
| `MILVUS_HOST` 不生效 | 容器中由 compose env 覆盖；本地用 `.env` 中的 |
| Tavily 提示未配 key | 正常降级 mock，配 key 后重启后端 |
| 前端 SSE 一直断 | 检查后端 CORS / nginx buffering（生产）|
| `pnpm dev` 占 3300 | `Stop-Process` 杀掉 |
