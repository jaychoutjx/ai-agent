# 01 · 项目概览

> 5 分钟搞清楚这个项目"是什么、解决什么、做到什么程度"。

---

## 一、一句话介绍

> **基于 LangChain + LangGraph 构建的企业级智能知识库问答系统，支持文档上传后的 RAG 问答和 Agent 自主推理两种模式。**

---

## 二、解决什么问题

### 痛点

企业内部沉淀了大量文档（产品手册、SOP、合同、技术规范），员工需要查资料时：

1. 关键词搜索召回率低，精确匹配但不懂语义
2. 直接问 ChatGPT 不能回答企业专属知识，且会"胡说八道"（幻觉）
3. 散落在 Confluence / Notion / 钉盘里，跨系统检索困难

### 解决方案

把企业文档变成 LLM 的"外脑"：

```
[文档库] ──[切片]──> [向量库] ──[检索]──> [Top-K 片段] ──[拼到 Prompt]──> [LLM 生成]
                                                                              │
                                                                              ↓
                                                                        [带引用的回答]
```

- **答案准确**：引用来源可追溯（点击溯源到原文段落）
- **拒答幻觉**：检索不到就直接说"不知道"，不瞎编
- **统一入口**：上传一次，跨文档问答

## 三、能做什么（功能清单）

### 3.1 三种交互模式

| 模式 | 触发 | 用途 | 示例 |
|------|------|------|------|
| **Chat** | 不带文档 | 通用闲聊 / 写作 | "帮我写一首七言绝句" |
| **RAG** | 上传文档后 | 文档问答 | "这份合同里违约金条款是什么" |
| **Agent** | 选 Agent 模式 | 多步推理 + 工具调用 | "查一下今年北京 GDP，并算它占全国百分之几" |

### 3.2 文档管理

- 支持格式：PDF（PyMuPDF）、DOCX（python-docx）、Markdown、TXT
- 单文件上限：50 MB
- 操作：上传 / 列表 / 删除（自动联动 Milvus + BM25 索引）
- 入库流水线：解析 → 切片（500 字符，80 重叠）→ 批量 Embedding（每批 10）→ 入 Milvus

### 3.3 RAG 高级特性

| 特性 | 默认 | 说明 |
|------|------|------|
| 向量检索 | ✅ 开 | text-embedding-v3 + Milvus HNSW |
| BM25 关键词检索 | ✅ 开 | jieba 中文分词，专有名词命中 |
| RRF 融合 | ✅ 开 | k=60，免归一化合并多路 |
| Reranker | ✅ 开 | gte-rerank-v2 Cross-Encoder 精排 |
| Multi-Query 改写 | ⚪ 可选 | LLM 生成 N 个语义变体 |
| HyDE 改写 | ⚪ 可选 | 先生成假设答案再检索 |

### 3.4 Agent 工具集

| 工具 | 来源 | 用途 |
|------|------|------|
| `search_knowledge_base` | 内部 RAG | 查私有知识库 |
| `web_search` | Tavily API | 联网搜实时信息 |
| `calculator` | 沙箱 eval | 算数学题 |
| `get_current_time` | 系统时钟 | 拿当前时间 |

> 没配 Tavily Key 时 `web_search` 会**优雅降级**返回 mock 文案，不会让整个 Agent 崩。

### 3.5 流式输出

- Chat / RAG：LLM token 级 SSE
- Agent：节点级 SSE（思考 → 工具调用 → 结果 → 生成），前端 `AgentTrace` 组件实时展示

## 六、典型使用场景

### 场景 1：客服知识库

上传 SOP 文档 → 客服问"用户投诉商品坏了怎么处理" → 系统给出步骤并引用 SOP 段落。

### 场景 2：法务合同审查

上传合同 → 问"这份合同里违约责任如何约定" → 系统定位到具体条款。

### 场景 3：技术文档助手

上传 API 文档 → 开发者问"如何调用 X 接口" → 系统给出参数和示例代码。

### 场景 4：Agent 多步任务

```
User: 查一下昨天 A 股大盘走势，然后帮我估算如果我有 1 万元，按沪深 300 涨跌算今天会变多少？

Agent:
  1. [web_search] "昨天 A 股大盘走势" → 拿到沪深 300 涨跌幅 +1.23%
  2. [calculator] "10000 * 1.0123" → 10123
  3. 综合回答："昨天沪深 300 涨 1.23%，您的 1 万元会变成 10123 元，涨 123 元。"
```
